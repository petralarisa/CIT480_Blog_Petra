Dangerous LINUX Commands

This week I was focusing on reviewing important LINUX commands and I stumbled upon an article about some dangerous commands to avoid. So in this blog I will talk about some of them, what those command do and why are they dangerous.


sudo rm -rf /*


Okay so this command is probably one of the deadliest commands. Let's break down this command:

  • sudo is used to give user the ability to run a command with the security privillege of another user which usually is the superuser's privillege
  • rm this command stands for remove which means to remove a file specified after this command
  • -rf the use of dash '-' in here is to specify flags. In this case we are using the r and f flags. The r flag stands for recursive meaning that it will continue performing the same operation to everything inside the files until there's no more subfile. The f flag stands for force, which means to force the operation and ignore he warnings or precautions.
  • /* this part specifies the path of the file. The '/' symbol means root and the '*' symbol is the wildcard symbol. So this part is pretty much refering to all files there is.


From the breakdown of the command, we can see that what this command would do is literally deleting everything inside the file system. Since it will delete evey files all the way from the root directory (the root directory is the very beginning of the file system), we will have nothing left.



mv / /dev/null


This command does similar thing as the previous command. Let's break down this command:

  • mv this command stands for move which is used
  • / this is the root directory
  • /dev/null is the null file. Anything written to it is discarded
