LINUX File Systems

In LINUX, everything is pretty much files. In organizing all these files, there is always a directory which holds files and also other sub-directories. The way it is managed can be illustrated as a tree structure. There’s the top level directory which is called the root directory. This root directory is usually represented by the forward slash symbol or ‘/’. Right under root, there are some main subdirectories that are just one level under the root directories and each of these directories contains some special files and has a specific content. You can see the illustration of the most common LINUX file system layout.





Here are the specific contents of each of these subdirectories of the root directory* :

  • /bin consists of common programs shared by the system, system administrator, and also the user
  • /boot consists of the startup files in the kernel. It is also where you can find the GRand Unified Bootloader (GRUB)
  • /dev contains the reference to all of the CPU peripheral hardwares
  • /etc Most important system configuration files are in /etc, this directory contains data similar to those in the Control Panel in Windows
  • /home is the Home directories of the common users.
  • /initrd (on some distributions) is the Information for booting. Do not remove!
  • /lib contains the Library files, includes files for all kinds of programs needed by the system and the users.
  • /lost+found Every partition has a lost+found in its upper directory. Files that were saved during failures are here.
  • /home is the Home directories of the common users.
  • /misc is the directories for miscellaneous purposes.
  • /mnt is the Standard mount point for external file systems
  • /net is the Standard mount point for entire remote file systems
  • /opt contains extra and third party software.
  • /proc is a virtual file system containing information about system resources.
  • /root The administrative user's home directory. Mind the difference between /, the root directory and /root, the home directory of the root user.
  • /sbin Programs for use by the system and the system administrator.
  • /tmp Temporary space for use by the system, cleaned upon reboot, so don't use this for saving any work!
  • /usr contains Programs, libraries, documentation etc. for all user-related programs.
  • /var stores all variable files and temporary files created by users, such as log files, the mail queue, the print spooler area, space for temporary storage of files downloaded from the Internet, or to keep an image of a CD before burning it.

*reference from: click here


Some useful commands relating to file systems:

  • pwd: this command is to show the path of your current working directory
  • ls: this command is to list all files within the directory you’re working on right now
  • cd: this command is to change directory and is usually followed by the path of the directory you want to change to
