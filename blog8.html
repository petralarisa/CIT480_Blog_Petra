Setting Up Kubernetes on Mac

I have decided that I want to learn kubernetes. So before I can get hands-on and play around with it I need to know what it is and how to prepare the environment needed.


What is Kubernetes?


Kubernetes is a platform for working with containers like docker. The word 'working' in here refers to deployment, scaling, and monitoring. In kubernetes we have a master node and this node is a part of a cluster, meaning that this master node is aware of other servers within the clusters that we have created and we can use to deploy the containers. During deployment, we are going to tell kubernetes what image do we want to use as the base and we can also add some other configuration which then will be the deployment (our application). Kubernetes will keep track of all of the configurations we set during deployment. This deployment has a deployment controller. What this does is to monitor, like when our application goes down, kubernetes will know about it and it will try to 'autoheal' by spinning another container and recover for itself. In terms of scaling, kubernetes has the ability to figure out where to put our deployment when we need to scale up.


What do we need to install before playing around with kubernetes?


Before we start installing some tools, we need to make sure that virtualization is supported on our machine. For mac, you can do it by using the command: sysctl -a | grep -E --color 'machdep.cpu.features|VMX'

If after you run the command successfully you receive an output that has VMX on it then it means that virtualization feature is enabled on your machine.

So now let's go to the list of tools we need to install.


  • kubectl
  • hypervisor
  • minikube

Installing kubectl on mac through terminal

Below are the list of commands I used. It is written in order.

  • curl -LO "https://storage.googleapis.com/kubernetes-release/release/$(curl -s https://storage.googleapis.com/kubernetes-release/release/stable.txt)/bin/darwin/amd64/kubectl"
  • chmod +x ./kubectl --> this command is used to make it excecutable
  • sudo mv ./kubectl /usr/local/bin/kubectl --> moving it to my PATH
  • kubectl version --client --> to confirm if the installation was successfully done

Installing minikube on mac through terminal

Since I already have hypervisor installed, I didn't need to install it again. In my case, I have vmware fusion. If you want to get it for free, you can download and install it using your CSUN account on CSUN imagine. We have the perks as CSUN student to get it for free.

Below are the list of commands I used to install minikube. It is written in order.

  • brew install minikube --> I used brew because I have homebrew installed. If you don't want to use homebrew, you may use the next command.
  • curl -Lo minikube https://storage.googleapis.com/minikube/releases/latest/minikube-darwin-amd64 \ && chmod +x minikube --> this command is used as an alternative to using homebrew as seen in the previous command
  • sudo mv minikube /usr/local/bin --> if you want to move it to your PATH
  • minikube start --driver=vmwarefusion --> testing by starting a kubernetes cluster. I put vmwarefusion because I used vmware fusion for my hypervisor.
  • minikube status --> this should output information about the host, kubelet, apiserver, and kubeconfig.
  • minikube stop --> to stop the cluster
